# 钢铁行业定量指标统一打分公式设计方案

## 一、统一评分框架

### 1.1 核心规则

所有指标（行业维度 & 自身维度）统一使用 **Winsorized Min-Max 归一化**，映射到 **[1, 10]** 区间：

- **P0 ~ P10 范围内的值** → 得 **1 分**（底部分位数）
- **P90 ~ P100 范围内的值** → 得 **10 分**（顶部分位数）
- **P10 ~ P90 之间的值** → 线性插值，得分 ∈ [1, 10]
- **空值（未披露）** → 得 **0 分**

### 1.2 通用公式

#### "越高越好"指标（占比型）：
```
S = clip(1 + 9 × (x - P10) / (P90 - P10), 1, 10)
```

#### "越低越好"指标（强度型）：
```
S = clip(1 + 9 × (P90 - x) / (P90 - P10), 1, 10)
```

其中：
- `x` = 当期值（行业维度）或改进率（自身维度）
- `P10` = 第10百分位数（底部分位）
- `P90` = 第90百分位数（顶部分位）
- `clip(v, 1, 10)` = 将值限制在 [1, 10] 区间

### 1.3 维度权重

| 指标类型 | 行业维度（当期数据） | 自身维度（改进数据） | 总分公式 |
|----------|---------------------|---------------------|----------|
| 通用指标 | 70% | 30% | 总分 = S_ind × 0.7 + S_self × 0.3 |
| 稳就业促就业 | — | — | 总分 = S_rate（仅基于年度变化率） |

> **稳就业特殊说明**：绝对就业人数不可跨企比较（宝武23.7万人 vs 凌钢0.35万人），因此仅使用年度变化率作为唯一评分维度，不做70/30拆分。

### 1.4 空值处理

| 情况 | 处理方式 | 总分影响 |
|------|----------|----------|
| 行业维度有值、自身维度空值（仅披露当期） | S_self = 0 | 总分 = S_ind × 0.7 + 0 × 0.3 |
| 行业维度空值、自身维度有值 | S_ind = 0 | 总分 = 0 × 0.7 + S_self × 0.3 |
| 两维度均空值 | S_ind = 0, S_self = 0 | 总分 = 0 |
| 稳就业空值 | S_rate = 0 | 总分 = 0 |

## 三、各指标完整打分公式

### ① 物理碳排放

**行业维度（70%）**— 当期物理碳强度，越低越好 ↓
```
S_ind = clip(1 + 9 × (2.1285 - x) / 0.5894, 1, 10)
```
> x = 当期物理碳强度（万吨CO₂/万吨产量）

**自身维度（30%）**— 改进率，越高越好 ↑
```
S_self = clip(1 + 9 × (r + 0.0629) / 0.1214, 1, 10)
```
> r = 改进率 = (基准期碳强度 - 当期碳强度) / 基准期碳强度

**总分**：
```
总分 = S_ind × 0.7 + S_self × 0.3
```

## 四、公式统一逻辑说明

### 4.1 为什么统一用 Winsorized Min-Max？

| 优势 | 说明 |
|------|------|
| **一致性** | 行业维度和自身维度使用完全相同的归一化逻辑，易于理解和维护 |
| **自适应** | P10/P90由数据分布自动决定，无需手动设上下限参数 |
| **抗outlier** | 极端值自动截断到1或10，不会扭曲中间80%企业的梯度 |
| **梯度合理** | 中间区间线性插值，相邻企业得分差与实际差距成正比 |
| **计算便捷** | 只需知道P10和P90两个参数，一行公式即可计算 |

### 4.2 为什么 P0-P10 得1分而非0分？

- **1分 = "有数据但表现最差"**，**0分 = "无数据"**
- 区分"披露了但差"和"不披露"，鼓励透明度
- 避免"最差企业"与"未披露企业"获得相同分数，丧失评分区分度

### 4.3 自身维度改进率的方向处理

无论原始指标是"越低越好"还是"越高越好"，改进率/提升率的评分方向**始终为越高越好**：

| 原指标方向 | 改进率定义 | 改进率含义 |
|-----------|-----------|-----------|
| 越低越好（强度型） | r = (基准 - 当期) / 基准 | r > 0 表示强度下降（改善） |
| 越高越好（占比型） | r = (当期 - 基准) / 基准 | r > 0 表示占比上升（改善） |

因此自身维度统一使用 `↑越高越好` 公式：
```
S_self = clip(1 + 9 × (r - P10) / (P90 - P10), 1, 10)
```

### 4.4 稳就业为什么不做70/30拆分？

| 考虑因素 | 分析 |
|----------|------|
| 行业维度（绝对就业数） | 宝武23.7万人 vs 凌钢0.35万人，量级差68倍，不可比 |
| 自身维度（变化率） | 年度变化率本身就是"相对改进"指标 |
| 逻辑冲突 | 若强行拆分，行业维度应排名绝对人数，但这恰恰不可比 |
| 结论 | 稳就业的评分 = 变化率的Winsorized打分，逻辑最自洽 |

## 六、数据缺失的影响量化

以物理碳排放为例（14家有完整数据，6家未披露）：

| 披露情况 | S_ind | S_self | 总分 |
|----------|-------|--------|------|
| 两期完整 | 正常1~10 | 正常1~10 | 1~10 |
| 仅披露当期 | 正常1~10 | 0 | 最高7 |
| 仅披露基准期 | 0 | 无法计算 | 0 |
| 两期均未披露 | 0 | 0 | 0 |

> **设计含义**: 未披露企业最高只能得7分（行业维度满分×0.7），惩罚力度适中，鼓励全量披露。

## 七、Excel 计算模板建议

### 7.1 行业维度计算列（适用于所有指标）

假设数据在列B（当期值），P10和P90为固定参数：

**越低越好指标（A类）**：
```excel
=MIN(10, MAX(1, 1 + 9 * (P90_cell - B_cell) / (P90_cell - P10_cell)))
```

**越高越好指标（B类）**：
```excel
=MIN(10, MAX(1, 1 + 9 * (B_cell - P10_cell) / (P90_cell - P10_cell)))
```

**空值处理**：
```excel
=IF(B_cell="", 0, MIN(10, MAX(1, 1 + 9 * (...) / (...))))
```

### 7.2 自身维度计算列

改进率数据在列H，方向统一为"越高越好"：
```excel
=IF(H_cell="", 0, MIN(10, MAX(1, 1 + 9 * (H_cell - P10_self) / (P90_self - P10_self))))
```

### 7.3 总分计算列

**通用指标**：
```excel
= S_ind_cell * 0.7 + S_self_cell * 0.3
```

**稳就业**：
```excel
= S_rate_cell  （无70/30拆分）
```
